# Инструкция пользователя

## 1. Назначение системы

`Ecookna QualAnalize` предназначена для проверки заказов на оконные конструкции по PDF-выгрузке. Система помогает:

- подобрать допустимые формулы стеклопакета по таблице слипания;
- проверить PDF-заказ на соответствие правилам контроля;
- показать замечания по конкретным позициям заказа.

Система поддерживает два пользовательских канала работы:

- Telegram-бот для проверки PDF;
- веб-интерфейс для подбора формул и проверки PDF.

## 2. Основные принципы работы

Система работает по следующим принципам:

- исходным документом является PDF-файл заказа;
- из PDF извлекаются позиции, размеры, формулы, раскладка и дополнительные признаки;
- для каждой позиции выполняются детерминированные проверки;
- результаты проверки сохраняются в базе и возвращаются пользователю в понятном текстовом виде.

Ключевой принцип: решение о наличии ошибки принимается не вручную и не по свободной интерпретации, а по формализованным правилам, реализованным в коде и SQL-процедурах.

## 3. Веб-интерфейс

Веб-интерфейс открывается по корневому адресу сервиса, обычно `http://<host>:8000/`.

На главной странице доступны два рабочих блока.

### 3.1. Блок "Подбор формулы"

Назначение: подобрать допустимые формулы по размерам стеклопакета.

Элементы интерфейса:

- поле `Ширина, мм`;
- поле `Высота, мм`;
- кнопка `Подобрать формулы`.

Что показывает результат:

- исходный размер;
- размер после округления;
- формулы из таблицы слипания;
- варианты формул по камерам;
- суммарную толщину формулы, если она рассчитана.

### 3.2. Блок "Проверка заказа"

Назначение: проверить PDF-файл заказа.

Элементы интерфейса:

- зона drag-and-drop для PDF;
- кнопка `Выбрать PDF`;
- индикатор выбранного файла;
- кнопка `Проверить файл`.

Важно: в интерфейсе явно указано, что корректная выгрузка должна быть получена из StartОкна: `Печать/Резерв/3.4 Заполнения`.

### 3.3. Окно результатов

После подбора формул или проверки PDF открывается модальное окно `Результат проверок`.

Элементы окна:

- бейдж режима работы: `Подбор формулы` или `Проверка PDF`;
- кнопка `Скопировать`;
- карточки с результатом или ошибкой;
- детальный список замечаний по позициям.

## 4. Порядок работы: подбор формулы

1. Откройте веб-интерфейс.
2. В блоке `Подбор формулы` укажите ширину и высоту стеклопакета в миллиметрах.
3. Нажмите `Подобрать формулы`.
4. Ознакомьтесь с результатом в модальном окне.

Результат может быть двух типов:

- правило найдено;
- правило не найдено для указанного размера.

Если правило найдено, пользователь видит:

- исходный размер;
- округленный размер;
- строку `Формулы из таблицы слипания`;
- допустимые формулы для 1- и 2-камерных пакетов;
- расчет общей толщины формулы в скобках.

## 5. Порядок работы: проверка PDF через веб-интерфейс

1. Подготовьте PDF-выгрузку заказа из StartОкна: `Печать/Резерв/3.4 Заполнения`.
2. Откройте веб-интерфейс.
3. В блоке `Проверка заказа` перетащите PDF в область загрузки или выберите файл кнопкой `Выбрать PDF`.
4. Нажмите `Проверить файл`.
5. Изучите результат в окне `Результат проверок`.

Возможные результаты:

- замечаний не найдено;
- файл не удалось корректно разобрать;
- по отдельным позициям найдены ошибки.

Если замечания найдены, для каждой проблемной позиции отображаются:

- номер позиции;
- размер;
- формула;
- направление открывания;
- раскладка;
- список ошибок.

## 6. Порядок работы: проверка PDF через Telegram-бота

1. Откройте чат с ботом.
2. При необходимости используйте `/start` или `/help`.
3. Отправьте PDF-файл заказа сообщением в чат.
4. Дождитесь завершения обработки.
5. Получите текстовый отчет по файлу.

Бот возвращает:

- сообщение об успешной проверке без ошибок;
- сообщение о невозможности извлечь позиции;
- отчет с перечислением проблемных позиций и найденных несоответствий.

## 7. Интерпретация результатов

Типовые статусы и реакции:

- `Ошибок не обнаружено`: позиция или файл прошли проверку;
- `Правило не найдено`: для округленного размера в таблице нет записи;
- `Найдены замечания`: есть хотя бы одна проблемная позиция;
- `Нужна корректная выгрузка`: PDF не соответствует ожидаемому формату.

Практическая рекомендация:

- если система не извлекла позиции, нужно повторно сформировать PDF в корректной форме;
- если найдено несоответствие по позиции, необходимо сравнить фактическую формулу с допустимой формулой из таблицы слипания и скорректировать заказ.

## 8. Ограничения пользовательского режима

- поддерживаются только PDF-файлы;
- корректность разбора зависит от структуры исходной выгрузки;
- проверка одиночного стекла без дистанционной рамки не запускает контроль слипания;
- веб-интерфейс показывает ограниченный набор справочной информации, а полная техническая детализация хранится в базе.

## 9. Что считать правильным исходным файлом

Для веб-интерфейса и Telegram-бота корректным считается PDF:

- содержащий позиции заказа в табличном виде;
- включающий номер позиции, формулу, размеры, площадь и массу;
- выгруженный из формы StartОкна `Печать/Резерв/3.4 Заполнения`.

Если файл отличается по структуре, система может не извлечь позиции или вернуть неполный результат.
